# IPTVScanner Refactoring Plan

## Goal

Refactor IPTVScan (flat 3-class project) into IPTVScanner, adopting IPTVManager's modular layered architecture. Preserve all existing features while improving structure and code organization.

## Design Decisions

- **No database** ‚Äî keep scan results in-memory (QAbstractTableModel), matching current behavior
- **No setup dialog** ‚Äî auto-use default config path at `QStandardPaths::AppConfigLocation/IPTVScanner.ini`
- **Keep existing I/O formats** ‚Äî URL template input and `widthxheight(fps),url` output format unchanged
- **No new export formats** ‚Äî keep current TXT save-as behavior

## Implementation Steps

### Step 1: Project Scaffolding

Create directory structure and root CMakeLists.txt.

**Create `IPTVScanner/CMakeLists.txt`** ‚Äî mirrors IPTVManager pattern:
- CMake 3.19+, C++17, Qt6 auto-tools
- `include(cmake/Platform.cmake)` + `include(cmake/FindFFmpeg.cmake)`
- `find_package(Qt6 6.8 REQUIRED COMPONENTS Core Widgets LinguistTools)`
- `add_subdirectory()` for each module
- `qt_add_executable(IPTVScanner ...)` linking all `iptv_*` libs
- macOS bundle config, FFmpeg dylib copy, translations, tests option

**Copy from IPTVManager (no changes):**
- `cmake/Platform.cmake`
- `cmake/FindFFmpeg.cmake`
- `src/platform/platform_defs.h`
- `src/platform/CMakeLists.txt`
- `scripts/fix_ffmpeg_rpaths.sh`

**Move from IPTVScan:**
- `AppIcon.icns`, `AppIcon.iconset/`, `tv.png` ‚Üí `resources/`
- `Info.plist.in` ‚Üí root (update name to IPTVScanner)

### Step 2: Core Module (`iptv_core`)

**Create `src/core/CMakeLists.txt`:**
```cmake
add_library(iptv_core STATIC
    config/appconfig.cpp
    logging/logmanager.cpp
)
target_include_directories(iptv_core PUBLIC ${CMAKE_CURRENT_SOURCE_DIR})
target_link_libraries(iptv_core PUBLIC Qt::Core)
```

**Create `src/core/config/configkeys.h`** ‚Äî scan config keys:
- `SCAN_URL`, `THREAD_MAX` (8), `TIMEOUT` (500), `AUTO_STEP`, `USE_URL`, `MODEL_FILE_PATH`, `ADD_TS`, `SLOW_SCAN`
- Migrated from MainPage's `#define` constants

**Create `src/core/config/appconfig.h/cpp`** ‚Äî singleton QSettings:
- `init(configPath)` ‚Äî load settings file
- Typed accessors for each config key
- Replaces MainPage's manual `loadConfig()`/`saveConfig()`

**Create `src/core/logging/logmanager.h/cpp`** ‚Äî file logging:
- Categories: `lcScan`, `lcNet`
- Reuse from IPTVManager

### Step 3: Multimedia Module (`iptv_multimedia`)

**Create `src/multimedia/CMakeLists.txt`:**
```cmake
add_library(iptv_multimedia STATIC
    stream_info.cpp
    stream_probe.cpp
    ffmpeg_utils.cpp
)
target_include_directories(iptv_multimedia PUBLIC ${CMAKE_CURRENT_SOURCE_DIR} ${CMAKE_CURRENT_SOURCE_DIR}/..)
target_link_libraries(iptv_multimedia PUBLIC Qt::Core iptv_core FFmpeg::avformat FFmpeg::avutil)
```

**Create `src/multimedia/stream_info.h/cpp`:**
```cpp
namespace Iptv::Multimedia {
struct StreamInfo {
    QString url;
    int width = 0;
    int height = 0;
    int fps = 0;
    int sourceType = 0;
    bool success = false;
};
}
```

**Create `src/multimedia/ffmpeg_utils.h/cpp`** ‚Äî from ScanThread:
- `r2d(AVRational)` ‚Äî already in ScanThread
- `inferSourceType(int width)` ‚Äî map width to type
- `sourceTypeName(int type)` ‚Äî display name

**Create `src/multimedia/stream_probe.h/cpp`** ‚Äî refactored from ScanThread:
- FFmpeg C API logic preserved (avformat_open_input, avformat_find_stream_info, av_find_best_stream)
- Timeout support preserved
- Retry with timeout halving preserved (slowThread + firstFlag)
- Qt signals instead of friend-class:
  - `probeSucceeded(StreamInfo)`
  - `probeFailed(QString error)`

### Step 4: Logic Module (`iptv_logic`)

**Create `src/logic/CMakeLists.txt`:**
```cmake
add_library(iptv_logic STATIC
    url_builder.cpp
    scan_service.cpp
)
target_include_directories(iptv_logic PUBLIC ${CMAKE_CURRENT_SOURCE_DIR} ${CMAKE_CURRENT_SOURCE_DIR}/..)
target_link_libraries(iptv_logic PUBLIC Qt::Core iptv_core iptv_multimedia)
```

**Create `src/logic/url_builder.h/cpp`** ‚Äî from MainPage:
- `expand(template)` ‚Üí `QStringList` ‚Äî recursive `{}` expansion
- `expandRange(pattern)` ‚Üí `QStringList` ‚Äî `[start-end]` and `#`-separated values
- `getIpFromUrl(url)` ‚Üí `QString` ‚Äî from MainThread
- `expandFromFile(path)` ‚Üí `QStringList` ‚Äî from MainPage::createUrlFromFile
- Supports up to 3 nested `{}` groups
- Mixed notation: `1#3#[5-7]`

**Create `src/logic/scan_service.h/cpp`** ‚Äî refactored from MainThread:
- Uses `QSemaphore` for concurrency (replaces polling `while(count >= thMax) msleep(10)`)
- Manages `StreamProbe` instances
- IP dedup via `QHash<QString, bool>`
- Config: thMax, timeout, autoStep, slowThread
- Signals: `scanProgress`, `scanResult`, `currentUrlChanged`, `scanFinished`
- `start(urls)` / `stop()` / `isRunning()`

### Step 5: UI Module (`iptv_ui`)

**Create `src/ui/CMakeLists.txt`:**
```cmake
set(CMAKE_AUTOUIC ON)
set(CMAKE_INCLUDE_CURRENT_DIR ON)

add_library(iptv_ui STATIC
    mainwindow.cpp
    mainwindow.ui
    models/scan_result_model.cpp
    widgets/log_widget.cpp
)
target_include_directories(iptv_ui PUBLIC ${CMAKE_CURRENT_SOURCE_DIR} ${CMAKE_CURRENT_SOURCE_DIR}/..)
target_link_libraries(iptv_ui PUBLIC
    Qt::Core Qt::Widgets
    iptv_core iptv_multimedia iptv_logic
)
```

**Create `src/ui/mainwindow.h/cpp/ui`** ‚Äî refactored from MainPage:
- Inherits `QMainWindow`
- Uses `AppConfig` singleton (replaces manual QSettings)
- Uses `ScanService` (replaces direct MainThread)
- Uses `ScanResultModel` for results table
- Uses `UrlBuilder::expand()` (replaces local createUrl)
- Menu bar: File (Save, Exit), Help (About)
- Central widget: config panel + results/log area
- All existing controls preserved: URL input, radio buttons, thread count, timeout, fine scan, auto skip, auto add TS

**Create `src/ui/models/scan_result_model.h/cpp`** ‚Äî QAbstractTableModel:
- Columns: URL, Width, Height, FPS
- In-memory storage (no database)
- Append/clear interface

**Create `src/ui/widgets/log_widget.h/cpp`** ‚Äî log display:
- QTextBrowser wrapper
- append/clear

### Step 6: Main Entry Point

**Create `src/main.cpp`:**
```cpp
int main(int argc, char *argv[]) {
    QApplication app(argc, argv);
    app.setApplicationName("IPTVScanner");
    // Load translations
    // Init LogManager
    // Init AppConfig with default path
    MainWindow w;
    w.show();
    return app.exec();
}
```
- No Bootstrap/DbSetupDialog needed
- Auto-use `QStandardPaths::AppConfigLocation/IPTVScanner.ini`

### Step 8: Unit Tests

**Create `tests/CMakeLists.txt`:**
```cmake
enable_testing()
find_package(Qt6 REQUIRED COMPONENTS Test)
add_executable(iptvscanner_tests test_url_builder.cpp)
target_link_libraries(iptvscanner_tests PRIVATE Qt::Test iptv_logic iptv_core)
add_test(NAME iptvscanner_tests COMMAND iptvscanner_tests)
```

**Create `tests/test_url_builder.cpp`:**
- `expand("239.49.{0#1#8}")` ‚Üí 3 results
- `expand("{[1-3]}")` ‚Üí 1, 2, 3
- `expand("{08-10}")` ‚Üí 08, 09, 10 (zero-padded)
- Nested: `"{1#2}:{3#4}"` ‚Üí 4 results
- `getIpFromUrl("udp://239.49.0.1:6000")` ‚Üí "239.49.0.1"
- `getIpFromUrl("http://192.168.1.1:12345/udp/239.49.0.1:6000")` ‚Üí "239.49.0.1"

## Migration Summary

| IPTVScan | IPTVScanner | Module |
|----------|-------------|--------|
| `MainPage::loadConfig/saveConfig` | `AppConfig` singleton | core |
| `MainPage::createUrl/createList` | `UrlBuilder::expand` | logic |
| `MainPage::createUrlFromFile` | `UrlBuilder::expandFromFile` | logic |
| `MainPage` QWidget | `MainWindow` QMainWindow | ui |
| `MainThread` | `ScanService` | logic |
| `ScanThread::run()` | `StreamProbe::run()` | multimedia |
| `ScanThread::r2d()` | `FfmpegUtils::r2d()` | multimedia |
| Save button (raw .txt) | Save button (same format) | ui |
| Hardcoded FFmpeg paths | `FindFFmpeg.cmake` | cmake |
| (none) | Unit tests | tests |

## Verification Plan

1. **Build**: `cmake -B build -DBUILD_TESTS=ON && cmake --build build`
2. **Run tests**: `cd build && ctest`
3. **Manual test**: Launch IPTVScanner, enter `http://192.168.1.1:12345/udp/239.49.0.{[1-3]}:{6000#8000}`, start scan, verify:
   - URL expansion produces 6 URLs (3 IPs x 2 ports)
   - Multi-threaded scanning works
   - Progress updates display
   - Results show in log
   - Save exports `.txt` in `widthxheight(fps),url` format
   - Settings persist across restarts
