# 频道源扫描功能详细说明

本文档详细介绍 TVMC 中「频道源扫描」功能的使用方法和技巧。

## 📖 功能概述

频道源扫描用于批量检测 IPTV 流媒体地址的可用性。通过配置地址模板，软件会自动展开所有可能的地址组合，并使用 FFmpeg 逐一探测每个地址是否可用。

## 🖥️ 界面说明

### 扫描地址区域

| 控件 | 说明 |
|------|------|
| **URL 单选按钮** | 选择 URL 输入模式 |
| **文件 单选按钮** | 选择文件输入模式 |
| **输入框** | 输入 URL 模板或文件路径 |
| **... 按钮** | 文件模式下选择文件 |

### 扫描参数区域

| 参数 | 说明 | 建议值 |
|------|------|--------|
| **线程** | 并发扫描线程数 | 8-16 |
| **超时(ms)** | 单个地址探测超时时间 | 500-10000 |
| **自动跳过** | 发现相同IP已有成功地址后自动跳过后续 | 开启 |
| **添加TS** | 结果中添加"待整理 TS"前缀 | 关闭 |
| **精细扫描** | EIO错误时减半超时重试 | 关闭 |

### 按钮区域

| 按钮 | 功能 |
|------|------|
| **开始** | 开始扫描（会清空上次结果） |
| **停止** | 停止当前扫描 |
| **清空** | 清空扫描结果 |
| **保存** | 保存扫描结果到文件 |

### 进度显示区域

| 显示项 | 说明 |
|--------|------|
| **状态栏** | 当前正在扫描的URL |
| **进度** | 已扫描数/总数 |
| **统计** | 成功数, 失败数 |
| **进度条** | 扫描进度百分比 |

### 结果区域

显示扫描结果日志，包括：
- 扫描到的可用地址
- 分辨率和帧率信息
- 使用帮助提示

## 📝 URL 模板语法

### 基本规则

URL 模板使用花括号 `{}` 定义可变部分，支持三种表达方式：

#### 1. 零散值（用 `#` 分隔）

```
{值1#值2#值3}
```

**示例：**
```
{1#3#5}        → 1, 3, 5
{6000#8000}    → 6000, 8000
```

#### 2. 范围值（用 `[start-end]` 表示）

```
{[起始值-结束值]}
```

**示例：**
```
{[1-10]}       → 1, 2, 3, ..., 10
{[08-10]}      → 08, 09, 10（带前导零）
{[100-200]}    → 100, 101, ..., 200
```

#### 3. 混合使用

```
{值1#值2#[start-end]}
```

**示例：**
```
{1#3#[5-7]}    → 1, 3, 5, 6, 7
{6000#[8000-8010]} → 6000, 8000, 8001, ..., 8010
```

### 嵌套规则

最多支持 **3层花括号嵌套**，系统会自动展开所有组合。

**示例：**
```
http://192.168.1.1:12345/udp/239.49.{0#1#2}.{[1-255]}:{6000#8000}
```

展开为：
```
http://192.168.1.1:12345/udp/239.49.0.1:6000
http://192.168.1.1:12345/udp/239.49.0.1:8000
http://192.168.1.1:12345/udp/239.49.0.2:6000
http://192.168.1.1:12345/udp/239.49.0.2:8000
...
http://192.168.1.1:12345/udp/239.49.2.255:8000
```

## 🔧 使用场景

### 场景一：扫描局域网组播地址

**需求：** 扫描 239.49.0.x 网段的所有地址，端口 6000

**URL 模板：**
```
http://192.168.1.1:12345/udp/239.49.0.{[1-255]}:6000
```

**参数设置：**
- 线程：8
- 超时：10000（局域网）
- 自动跳过：开启

### 场景二：扫描多个端口

**需求：** 扫描 239.49.0.1 的多个端口

**URL 模板：**
```
http://192.168.1.1:12345/udp/239.49.0.1:{6000#8000#8008}
```

**参数设置：**
- 线程：4
- 超时：1000

### 场景三：使用文件批量扫描

**需求：** 从文件读取大量地址进行扫描

**步骤：**
1. 准备文本文件，每行一个地址模板
2. 选择 **「文件」** 单选按钮
3. 点击 **「...」** 选择文件
4. 点击 **「开始」**

**文件格式示例：**
```
http://192.168.1.1:12345/udp/239.49.0.{[1-100]}:6000
http://192.168.1.1:12345/udp/239.49.1.{[1-100]}:6000
http://192.168.1.1:12345/udp/239.49.2.{[1-100]}:6000
```

## ⚙️ 参数调优

### 线程数设置

| 场景 | 建议线程数 | 说明 |
|------|-----------|------|
| 局域网 + 高性能服务器 | 16-32 | 服务器能承受高并发 |
| 局域网 + 普通服务器 | 8-16 | 平衡速度和稳定性 |
| 外网 + udpxy | 4-8 | udpxy 连接数有限 |
| 不确定 | 8 | 默认推荐值 |

### 超时时间设置

| 场景 | 建议超时 | 说明 |
|------|---------|------|
| 局域网直连 | 500-1000 | 网络延迟低 |
| 局域网 + udpxy | 10000 | 需要等待 udpxy 响应，且组播组件有限制 |
| 外网 | 5000-10000 | 网络延迟高 |
| 不确定 | 10000 | 不做超时限制 |

### 自动跳过

**开启时：**
- 发现某个 IP 地址有成功的频道后，后续该 IP 的其他地址自动跳过
- 适用于每个 IP 只部署一个频道的场景
- 大幅提高扫描效率

**关闭时：**
- 扫描所有地址，不做跳过
- 适用于一个 IP 部署多个频道的场景

### 精细扫描

**开启时：**
- 遇到 EIO 错误时，自动减半超时时间重试
- 最多重试 2 次
- 适用于网络不稳定的场景

**关闭时：**
- 遇到错误直接标记失败
- 适用于网络稳定的场景

## 📊 结果格式

扫描成功的地址会以以下格式显示：

```
待整理 TS 720x576(25),http://172.28.0.2:25140/udp/239.49.0.1:8000
```

格式说明：
- `待整理 TS`：源类型（可通过「添加TS」选项控制）
- `720x576`：视频分辨率
- `(25)`：帧率
- `http://...`：完整地址

## 💾 保存与导入

### 保存扫描结果

1. 扫描完成后，点击 **「保存」** 按钮
2. 选择保存位置和文件名
3. 文件格式为 `.mc`（TVMC 专用格式）

### 导入到频道管理

1. 切换到 **「软件配置」** 标签页
2. 在 **「导入文件」** 分组中选择保存的 `.mc` 文件
3. 点击 **「导入」** 按钮
4. 导入成功后可在 **「频道编辑」** 标签页查看和编辑

### 导出为其他格式

导入后可以通过 **「导出文件」** 分组导出为：
- **M3U**：标准播放列表
- **TXT**：文本格式
- **XLS**：Excel 表格
- **CSV**：逗号分隔值

## ⚠️ 注意事项

1. **地址数量**：模板展开后地址数量可能很大，请注意控制范围
2. **服务器负载**：高并发扫描可能对服务器造成压力，请合理设置线程数
3. **网络环境**：根据实际网络环境调整超时时间
4. **结果保存**：扫描结果不会自动保存，请及时保存
5. **重复扫描**：开始新扫描会清空之前的结果

## 🐛 常见问题

### Q: 扫描结果为空？

**可能原因：**
- 地址模板配置错误
- 服务器不可达
- 超时时间太短

**解决方法：**
- 检查地址模板语法
- 测试服务器连通性
- 增加超时时间

### Q: 扫描速度很慢？

**可能原因：**
- 超时时间太长
- 线程数太少
- 网络延迟高

**解决方法：**
- 适当减少超时时间
- 增加线程数
- 检查网络环境

### Q: 扫描中断了怎么办？

**解决方法：**
- 点击「清空」清空结果
- 重新开始扫描
- 建议开启「自动跳过」提高效率

### Q: 如何提高扫描效率？

**建议：**
- 开启「自动跳过」
- 合理设置线程数
- 根据网络环境调整超时
- 使用文件模式批量扫描
